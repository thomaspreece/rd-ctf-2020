Rules of Engagement

Please hack responsibly and adhere to the rules. You are welcome to use any tools you have at your disposal but please avoid automatic scanning tools such as dirb or sqlmap as they may DDOS the server and ruin the game for others. None of the challenges have been designd to require automated scanning tools so you won't be at a disadvantage by not using them (and some challenges actively prevent them).
Only the following domains are VALID for hacking against:

  • Anything under *.ctf.rd-security-training.bbctest01.uk
  • juiceshop.training.bbc-security-champions.com

The following domains are NOT VALID for hacking against:

  • ctfd.rd-security-training.bbctest01.uk
  • Anything else under rd-security-training.bbctest01.uk or bbctest01.uk

Getting Help

If you are in R&D you can get help from the rd-ctf slack channel. Alternatively you can message tomp on slack directly. Also pay close attension to the challenge title and text within. The challenges are written in a way that key words are included to hint towards the solution.
These challenges are designed to stretch you so don't be put off if you don't get them immediately, I'll be on hand to offer out hints (and the challenges also have hints too!). Solutions to the challenges will be available after the competition.

New Challenges

Generally the point value of a challenge will show how difficult it is. A chunk of challenges are labelled "JuiceShop", these are using the same tool that has been used in previous hacking workshops so most of you should be familiar with it already. Because of this, these JuiceShop challenges are lower point values then the other unseen challenges. So you may actually find some higher point challenges easier to do then the JuiceShop ones.

Over the course of the competition new challenges will become available so if you are struggling with existing challenges check back later for some new ones that you may have more luck with.
